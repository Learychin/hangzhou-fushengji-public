# 杭州新闻与黑色幽默设计口径

## 目的

新闻要让玩家记住一段因果并据此交易。笑点服务于判断，不把真实纠纷当作猎奇素材，也不影射可识别的消费者、记者或商家。

## 可用结构

1. 页面规则、客服解释和实际执行互相不认识。
2. 所有入口都自称官方，直到真正官方出现。
3. 技术升级解决了旧问题，同时发明一个更现代的新问题。
4. 负责人一直在开会，解决方案也在排队等待解决。
5. 传闻先抬高需求，后续报道揭示代价，最终给出有限修复。

## 内容约束

- 使用虚构人物、合成场景和泛化机构，不复制真实当事人的身份、损失与原话。
- 每条链保持 `线索 -> 反转 -> 收尾`，每段间隔 2-3 天，并明确影响商品与方向。
- 玩家即使完全忽略新闻，也能完成核心交易；新闻只提供可利用的信息差。
- 同一天只让最强的一条进入弹窗，其余影响保留在行情中，防止信息疲劳。
- 笑点来自系统之间的错位，不嘲笑受损者，不把严重伤害、医疗纠纷或诈骗损失做成奖励。

## 已采用的合成链

- 湖滨排队：队伍先于原因出现，真相公开后队伍更长。
- 桌面搭子：新品走红、投诉反转、固件修复。
- 消费券规则：海报、客服和结算系统对规则各有理解。
- 官方维修搜索：搜索结果都自称官方、维修短暂生效、真正官方最后出现。

这些内容只借用杭州民生新闻常见的叙事机制，不复刻具体案件。
