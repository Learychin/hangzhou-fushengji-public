# 移动端真机验收清单

目标：确认当前移动端 Alpha 在真实手机上达到“单手顺畅玩完一局，约 10 分钟，有明确再开一局动力”。

## 启动真机预览

在仓库根目录运行：

```bash
node scripts/mobile_device_preview.mjs
```

脚本会：

- 启动一个局域网静态预览服务。
- 输出可在同一 Wi-Fi 手机上打开的地址。
- 输出手机验收辅助页 `/__mobile-check`，包含游戏入口、手机环境自检、清缓存后打开、必过清单和每局记录框。
- 在 `reports/manual_mobile_check/` 下生成一份可填写的验收记录，包含 iPhone/Android 各 3 局记录表。

手机打不开时，优先检查：手机和电脑是否同一 Wi-Fi、macOS 防火墙是否阻止 Node、本机是否开了 VPN/隔离网络。

手机上可以先打开辅助页，点“刷新自检”确认页面版本和 `main.js` 版本一致，再从辅助页进入游戏；如果手机疑似跑旧包，先点“清缓存后打开”。跑完一局后回到辅助页勾选清单并记下用时、分数和是否愿意再开。

跑完 6 局后，把辅助页生成的 Markdown 摘要保存到 `reports/manual_mobile_check/`，再运行：

```bash
node scripts/check_manual_mobile_report.mjs reports/manual_mobile_check/你的验收摘要.md
```

脚本会检查版本、两类手机各 3 局、8-12 分钟命中、主按钮流程、遮挡、刷新恢复、再开意愿和必过清单。

## 设备范围

- iPhone Safari：至少一台刘海屏或灵动岛机型。
- Android Chrome：至少一台 360-430 CSS px 宽度机型。
- 横竖屏：以竖屏为主；横屏不要求完整体验，但不能出现无法返回或按钮不可点。

## 安装与启动

- 直接浏览器打开页面，首屏无横向滚动。
- 辅助页自检显示当前版本一致；清缓存后打开仍能正常进游戏。
- 添加到主屏幕后启动，PNG 图标、标题、主题色正常。
- 首次打开后 service worker 可注册，弱网/短暂离线时页面壳可重新打开。
- 从主屏启动后底部主按钮不被系统安全区遮挡。
- 切到后台再回来，当前局状态不应错乱。
- 中途刷新页面或被系统回收后重开，能恢复到同一局进度。

## 单手流程

- 只用底部主按钮能完成一整局 45 天。
- 开局弹窗能看到本局目标或本机纪录，不需要先玩一局才知道要冲什么。
- 开局按钮能承接目标：首局是“开始 · 跟着下一步”，回访玩家是“开始 · 冲目标/刷新纪录”。
- 主按钮文案能自然切换：买入、卖出、还债、换站、结束。
- 目标条能看懂当前差距：先回正、下一档、连赚状态。
- 盈利后能立刻看到“距下一档”和连赚反馈。
- 接近升档或破纪录时，底部主按钮有明显冲刺高亮。
- 中途出现低现金或欠债压力时，玩家能知道下一步该做什么。
- 全程不需要精细点按小表格才能继续推进。

## 时间与节奏

- 普通玩家完整一局控制在 8-12 分钟。
- 自动推荐主流程点击数以 55-140 次为观察区间；不同隐藏节奏档位不强求相同。
- 连同新闻关闭、扩仓确认在内，总点击不超过 150 次，估算局长不超过 15 分钟。
- 当前五档自动长测总点击为 62-147 次、估算 6.2-14.7 分钟；五档各 3 局均完成 45 天。
- 底部主按钮推荐扩仓时应一键完成，不再弹出重复确认；主动点击仓位时仍应打开成本与管理费确认弹窗。
- 游戏结束后能一眼看到评级、下一局目标、下一局挑战、本局高光、本机生涯、本局徽章。
- “再来一局”按钮需要带明确目标，例如“再来一局 · 冲 ¥500,000”或“再来一局 · 破纪录”。

## 视觉与触控

- 360x740、390x844、430x932 附近宽度均无横向溢出。
- 底部主按钮、机会卡按钮、弹窗按钮触控高度不低于 44px。
- 结算弹窗内容可滚动，不遮挡“再来一局”。
- 键盘弹出输入昵称时，按钮仍可操作。

## 通过标准

- 两类手机各完整跑 3 局，没有卡死、错位、遮挡、无法继续。
- 至少 4/6 局落在 8-12 分钟。
- 玩家能复述“这一局/下一局要冲什么”，说明留存目标被看见。
